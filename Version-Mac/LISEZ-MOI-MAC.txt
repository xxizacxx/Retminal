============================================================
  RETMINAL — VERSION MAC  (comment fabriquer ton .dmg)
============================================================

IMPORTANT (a lire) :
- Un .dmg / une appli Mac NE PEUT se fabriquer QUE sur un Mac.
  C'est pour ca que tu fais le build toi-meme sur le Mac (1 double-clic).
- Cette version Mac n'a PAS pu etre testee (le code a ete adapte
  "a l'aveugle"). C'est une 1re version : si un truc bug sur le Mac,
  envoie l'erreur et on corrige.


------------------------------------------------------------
  ETAPE 1 — Mettre ce dossier sur le Mac
------------------------------------------------------------
Copie TOUT le dossier "Version-Mac" sur le Mac
(cle USB, AirDrop, Google Drive... comme tu veux).


------------------------------------------------------------
  ETAPE 2 — Avoir Python 3 (une seule fois)
------------------------------------------------------------
Le build a besoin de Python 3 AVEC Tkinter.
Le plus simple : telecharge "macOS 64-bit universal2 installer" ici :
   https://www.python.org/downloads/macos/
(celui de python.org inclut deja Tk, parfait.)


------------------------------------------------------------
  ETAPE 3 — Lancer le build
------------------------------------------------------------
⭐ METHODE QUI MARCHE A TOUS LES COUPS (recommandee) :
  Quand on transfere depuis Windows/iCloud, le Mac enleve la
  "permission de lancer" du fichier -> le double-clic ne marche pas.
  Le plus simple : passe par le Terminal (pas de permission a gerer).

  1. Ouvre l'app  Terminal  (Spotlight: tape "Terminal").
  2. Tape  cd  puis un ESPACE, puis GLISSE le dossier "Version-Mac"
     dans la fenetre du Terminal, et appuie sur Entree.
        (ca donne un truc comme : cd /Users/toi/.../Version-Mac )
  3. Tape :  bash build_mac.command   et appuie sur Entree.

  -> "bash" n'a PAS besoin de la permission. Ca lance le build direct.

AUTRE METHODE (double-clic) :
  Si tu preferes double-cliquer build_mac.command :
   - d'abord, dans le Terminal (etape 2 ci-dessus), tape une fois :
        chmod +x build_mac.command
   - ensuite tu peux double-cliquer.
   - si le Mac dit "developpeur non identifie" :
        clic-DROIT sur build_mac.command -> Ouvrir -> Ouvrir.

Ca telecharge les outils (PyInstaller...) et fabrique l'appli.
Patiente quelques minutes. A la fin tu auras :

        Retminal.dmg        <- l'installateur
        dist/Retminal.app   <- l'appli toute prete


------------------------------------------------------------
  ETAPE 4 — Installer
------------------------------------------------------------
Ouvre  Retminal.dmg , puis glisse  Retminal  dans  Applications.

  -> Au 1er lancement, le Mac peut bloquer l'appli (pas signee).
     Fais : clic-DROIT sur Retminal  ->  Ouvrir  ->  Ouvrir.


------------------------------------------------------------
  TES SECRETS (IP / mots de passe du VPS, token Claude)
------------------------------------------------------------
Ils sont dans le fichier  secret.env  (PAS ".env" : sur Mac un
nom qui commence par un point est CACHE dans le Finder ; "secret.env"
se voit). Mets tes infos dedans AVANT de fabriquer le .dmg.

Apres installation, l'appli range une copie modifiable ici :
  ~/Library/Application Support/Retminal/secret.env
(c'est la que tu peux le modifier une fois l'appli installee.)


============================================================
  CE QUI MARCHE / CE QUI EST LIMITE SUR MAC
============================================================
MARCHE (normalement) :
  - Le terminal (shells zsh / bash / sh au lieu de WSL/cmd)
  - L'editeur "carnet", la page "config", l'explorateur
  - Le SSH / VPS (connect, deploy, logs, moniteur serveur...)
  - Le markdown + couleurs §, l'aide a la saisie, les onglets

LIMITE / pas encore sur Mac :
  - "sysinfo" (infos du Mac lui-meme : CPU/RAM en direct)
    -> sur Mac ca affiche juste un message pour l'instant.
  - Les coins arrondis de la fenetre (detail Windows).
  - Le bouton "reduire" peut ne rien faire (fenetre sans bordure).

Si tu trouves un bug : note la commande + le message d'erreur
et donne-les moi, je corrige dans Version-Mac (jamais ton Windows).
